#### Бизнес правила:

- BR‑1. Слоты строятся с учётом длительности услуг.
- BR‑2. Записаться можно не позже чем за N часов до выбранного слота записи.
- BR‑3. В рамках одного временного слота у мастера не может быть более одной записи.
- BR‑4. В качестве защиты времени сотрудника должна быть предусмотрена предоплата.
- BR‑5. Запись считается подтверждённой только после успешной предоплаты.
- BR‑6. Повторное подтверждение платежа не создаёт вторую запись.
- BR‑7. Если клиент не пришёл, запись отменяется, предоплата не возвращается.
- BR‑8. Если клиент отменил запись, предоплата не возвращается.
- BR‑9. Если мастер отменил запись, предоплата возвращается.
- BR‑10. Если мастер хочет перенести запись, клиенту отправляется подтверждение переноса записи.
- BR‑11. Если мастер хочет перенести запись и клиент отправил подтверждение, клиенту предлагается N свободных окон для записи.
- BR‑12. Если клиент решает не продолжать запись, предложенную мастером после переноса, запись отменяется и предоплата возвращается.
- BR‑13. Если клиент отправил отказ на перенос записи, запись отменяется и предоплата возвращается.
- BR‑14. Если клиент не подтвердил предложение и не отправил отказ, запись за N часов до назначенного времени отменяется, предоплата не возвращается.
- BR‑15. Перед каждым визитом должно быть отправлено напоминание в виде сообщения за N часов.
- BR‑16. Отзыв можно оставить только после завершения визита.
- BR‑17. Один клиент после одного визита может оставить только один отзыв.
- BR‑18. Отзыв состоит из отметки от 1 до 5 и текста.
- BR‑19. Мастер может изменять статус, принимать записи или предлагать перенос и отмену только клиентам, которые записаны к нему.
- BR‑20. Мастер может загрузить в профиль не более N фото своих работ.
- BR‑21. Мастер может отменить не более N записей за месяц.
- BR‑22. Администратор салона может управлять акциями и просматривать записи только своего салона.
- BR‑23. Администратор создаёт профиль мастера и передаёт ему редактирование.
- BR‑24. Администратор может деактивировать профиль мастера в системе или возможность записи на определённую услугу по причине отсутствия мастера в заданном интервале.
- BR‑25. Деактивация профиля мастера или отдельных услуг запрещает новые записи к нему или по деактивированным для этого мастера услугам.
- BR‑26. Пользователь с ролью «мастер», действуя как клиент, не может записаться к себе самому; система исключает такой выбор или блокирует подтверждение записи .
