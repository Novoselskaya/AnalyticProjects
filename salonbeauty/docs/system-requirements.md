
### SR-01. Каталог, поиск и запись

Система обеспечивает поиск салона, мастера и услуги по предложенным городам и салонам города, отображение доступных услуг и мастеров для записи, построение расписания и слотов в расписании с учётом длительности услуг, оформление записи. Пользователям без учетной записи предоставляется ограниченный функционал, но возможность записи с регистрацией. Для авторизованного клиента — запись без повторной регистрации.

Критерии приёмки:
  - Пользователь может найти услугу/мастера в выбранном городе и перейти к выбору времени.
  - Система не отображает для новой записи деактивированных мастеров и услуг в объёме BR‑25.
  - Выбор времени учитывает занятость мастера (BR‑3) и минимальный горизонт записи (BR‑2).
  - Запись переводится в подтверждённое состояние только после успешной предоплаты (BR‑5).
  - Повторное подтверждение платежа не порождает вторую запись (BR‑6).

  Артефакты: UC‑NCL‑1, UC‑NCL‑2, UC‑NCL‑3, UC‑CL‑1, UC‑CL‑2, BR‑1 … BR‑6, BR‑25.


### SR-02. Учёт записей клиента: история, перенос и отмена со стороны клиента

Система хранит записи клиента, предоставляет просмотр истории, поддерживает перенос записи клиентом на другое окошко для записи, а также отмену записи по инициативе клиента с применением политики удержания предоплаты.

Критерии приёмки:
  - Клиент видит список своих записей и статусы, согласованные с доменной моделью.
  - Перенос отклоняется при нарушении правил слотов или времени (BR‑2, BR‑3).
  - При отмене клиентом предоплата не возвращается (BR‑8); статус записи отражает отмену.

Артефакты: UC‑CL‑3, UC‑CL‑5, UC‑CL‑6, BR‑1 … BR‑3, BR‑8.


### SR-03. Предоплата как условие подтверждения и обработка платежей

В системе предоплата является обязательной, тк обеспечивает защиту мастера от неиспользованного им времени. Система обеспечивает связь оплаты с записью,  идемпотентность операций регистрации оплаты при повторных уведомлениях от платёжного провайдера или повторных запросах клиента.

 Критерии приёмки:
  - Без успешного подтверждения предоплаты запись не считается подтверждённой (BR‑5).
  - Повторная подтверждение того же платежа не создаёт дубликата записи и не дублирует финансовый эффект (BR‑6).

Артефакты: UC‑CL‑2, UC‑CL‑4, UC‑NCL‑2, BR‑4 … BR‑6.
 

### SR-04. Перенос записи по инициативе мастера и согласование с клиентом

Система поддерживает сценарий, в котором мастер предоставляет перенос, в следствии чего клиент получает запрос на подтверждение участия или может отклонить предложение, после согласия отображается N свободных окон. Предусмотрены исходы с возвратом предоплаты при отказе, без возврата в случае не ответа клиента за N часов до записи, а также успешный перенос на новое время с сохранением предоплаты. 

Критерии приёмки:
  - Уведомление о необходимости реакции на перенос (BR‑10).
  - Отказа от предложения переноса и отказ от предлагаемых слотов приводят к отмене с возвратом (BR‑12, BR‑13).
  - Истечение срока без подтверждения и без отказа обрабатывается по BR‑14.
  - Мастер не может произвести перенос чужой записи (BR‑19), а также учитывается лимит отмен мастера (BR‑21).

Артефакты: UC‑CL‑9, UC‑CL‑10, UC‑M‑5, BR‑10 … BR‑14, BR‑19, BR‑21.


### SR-05. Расписание, страница и действия мастера в системе

В системе мастер просматривает расписание , изменяет статус записи, в том числе фиксирует неявку, загружает портфолио с ограничением объёма, отменяет запись клиента с возвратом предоплаты и инициирует перенос. Все операции ограничены записями данного мастера. Мастер может просматривать отзывы о себе. Также мастер, имея возможности зарегистрированного клиента,  не может записаться сам к себе.

Критерии приёмки:
  - Расписание отражает записи и свободные интервалы (BR‑1, BR‑3).
  - Неявка приводит к отмене записи без возврата предоплаты (BR‑7).
  - Отмена мастером инициирует возврат предоплаты (BR‑9) с учётом лимита (BR‑21).
  - Загрузка работ не превышает N файлов (BR‑20).
  - Выбор собственной карточки мастера или подтверждение такой записи блокируется или исключается на этапе записи (UC‑CL‑1, UC‑CL‑2).

  
Артефакты: UC‑M‑1 … UC‑M‑6, BR‑7, BR‑9, BR‑15, BR‑19 … BR‑21, BR‑26; UC‑CL‑1, UC‑CL‑2.


### SR-06. Отзывы после визита

В системе после завершения визита клиент может оставить один отзыв на визит, который  содержит оценку по шкале 1–5 и текст. Оставить отзыв до завершения записи нельзя. Отзывы не доступны для просмотра для незарегистрированных клиентов.

Критерии приёмки:
  - Система отклоняет создание второго отзыва на тот же визит (BR‑17).
  - Данные отзыва проверяются по BR‑18.
  
Артефакты: UC‑CL‑7, UC‑CL‑8, UC‑M‑6; BR‑16 … BR‑18.


### SR-07. Администратор салона

Администратор управляет только своим салоном: просмотр записей и профилей мастеров, редактирование данных, назначение акций на услуги, финансовая статистика по записям салона, создание профиля мастера с передачей права редактирования мастеру, деактивация мастера или услуги в заданном интервале . Деактивация запрещает новые записи на затронутые сущности. Имеет все возможности зарегистрированного клиента.

Критерии приёмки:
  - Любая операция администратора проверяется на принадлежность к салону администратора (BR‑22).
  - Созданный администратором профиль мастера становится доступен мастеру для самостоятельного ведения согласно (BR‑23).
  - После деактивации новые записи на деактивированные сущности невозможны (BR‑24, BR‑25).
  
Артефакты:  UC‑A‑1 … UC‑A‑6 , BR‑22 … BR‑25.


### SR-08. Владелец сети

Владелец имеет все возможности администратора и получает сводные отчёты по сети салонов: статистика по записям и оплат за период.
Критерии приёмки:
  
  - Отчётные показатели, собранные по своему салону.
 
Артефакты: UC‑O‑1.
