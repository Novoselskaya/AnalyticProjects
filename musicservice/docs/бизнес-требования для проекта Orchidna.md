# Платформа агрегации музыкальных сервисов  

## 1. Общее описание 

Реализуется приложение для агрегации музыки из различных источников (Яндекс, VK, Spotify), а также с дополнительным сервисом Orchidna для публикации собственных треков и авторов.

**Цель продукта:** 
1. Для пользователя: 
    - дать пользователю единый интерфейс для поиска, прослушивания и организации музыки из нескольких внешних сервисов (Яндекс, VK, Spotify) по более низкой цене и за счет общей подписки
    - собственный сервис Orchidna с уникальными новыми авторами и треками
    - возможность создавать свои плейлисты, делиться ими в общем доступе.

2. Для автора:
    - создавать свой контент в виде альбомов с треками
    - развивать свой профиль как автора музыки с подписками и прослушиваниями. 

3. Для модератора: 
    - просматривать и обрабатывать заявки на роль автора
    - модерировать контент (блокировать пользователей, авторов, треки)

**Бизнес-цели платформы:**
- Подключение адаптеров
- Развитие собственного каталога Orchidna.
- Поддержка различных ролей и соблюдение правовых ограничений
- Модерация контента и заявок на статус автора
- Дедубликация треков

### Основные функции

- Поиск музыки по всем подключенным внешним сервисам
- Дедупликация треков с одинаковым названием 
- Создание плейлистов из треков разных сервисов
- Сохранение, удаление треков(из плейлистов и с платформы)
- Публикация собственного контента в Orchidna
- Управление пользователями и их данными
- Разделение ролей пользователей

## 2. Роли пользователей

### 2.1 Анонимный пользователь

**Аноним может:**
- Поиск музыки в выбранных сервисах
- Прослушивание треков (может быть ограниченное)
- Просмотр информации о треках и авторах
- Просмотр публичных плейлистов (может быть будут плейлисты допустим топ 10-самых популярных)  

**Аноним не может:**
- Создавать плейлисты
- Сохранять песни, добавлять в плейлист 
- Получить доступ к Orchidna

### 2.2 Зарегистрированный пользователь

**Возможности:**
- Все возможности наследует от анонимного пользователя
- Создание плейлистами и управление ими
- Сохранение треков в том числе и в плейлист
- История прослушиваний(последние 20)(плейлист недавно прослушанные)
- Подписка на авторов
- Доступ к Orchidna
- Сохранять чужой плейлист


### 2.3 Автор

**Возможности:**
- Все возможности зарегистрированного пользователя
- Публикация собственных треков в Orchidna
- Управление опубликованными треками (удаление, редактирование)
- Просмотр статистики прослушиваний своих треков
- Управление профилем автора

### 2.4 Модератор

**Возможности:**
- Все возможности зарегистрированного пользователя
- Просмотр и обработка заявок на роль автора
- Модерация контента (треки возможно определенные)
- Блокировка пользователей и авторов

### 3.1 Треки  

3.1.1 Поиск треки
Поиск музыки осуществляется любым пользователем. 
 Поиск по всем сервисам
- Пользователь вводит запрос
- Система отправляет запрос в агрегатор 
- Агрегатор отправляет запрос для всех адаптеров
- Агрегатор получает результаты и выполняет дедупликацию
- Пользователю показывается объединенный список с указанием источника

 Поиск по конкретному сервису
- Пользователь выбирает фильтр по сервису или сервисам
- Запрос отправляется через агрегатор в выбранный адаптер/ры в выбранный сервис/сы
- Пользователю показывается объединенный список или 1 трек

3.1.2 Сохранение трека в плейлист
- Пользователь (зарегистрированный) или автор могут сохранить трек в плейлист(общий плейлист музыки или определенный плейлист)

3.1.3 Редактирование трека(автором)
- Автор до публикации альбома может отредактировать трек. 

3.1.4 Удаление трека(автором)
- Автор может удалить трек из альбома до и после публикации.
- После публикации используется концепция мягкого удаления. при удалении трек сначала становится неактивным, а позже(через неделю) удаляется навсегда.


### 3.2 Плейлисты

Плейлисты бывают публичными и личными. Автор плейлиста(пользователь) может сделать свой плейлист видимым, а также видимым свое имя. 
3.2.1 Создание плейлиста
- Пользователь(зарегистрированный) создает новый плейлист с названием (можно пустой)
- Сохраняет плейлист 
- Добавляет треки 

3.2.2 Удаление плейлиста 
- Пользователь может удалить плейлист или закрыть его от других

3.2.3 Редактирование плейлиста
- Пользователь может отредактировать только свой плейлист (добавить музыку, поменять местами треки, обновить картинку либо название, сделать видимым/невидимыми

**Автоматические плейлисты** 
Плейлисты, которые формирует система
- Недавно прослушанные - последние 20-30 треков, которые попадут туда после 60% прослушания
- Обновляются автоматически 

### 3.3 Публикация контента (Автор)

**Загрузка трека**
- Автор создает альбом( не пустой, минимум один трек)
- Заполняет метаданные (название, автор, альбом(даже с 1 аудиофайлом), жанры, обложка)
- Трек проходит модерацию (нельзя один и тот же трек, без дубликатов)
- После одобрения публикуется в сервисе Orchidna
- Отложенный релиз плейлиста(в определенное время от автора)

 **Редактирование трека**
- Автор изменяет метаданные
- Изменения сохраняются
- Трек остается доступным

**Удаление трека**
- Автор удаляет трек
- Трек удаляется из каталога Orchidna
- Трек удаляется из всех плейлистов пользователей (или помечается как недоступный)
 

### 3.4 Регистрация и авторизация

**Регистрация пользователя**
- Пользователь регистрируется
- Получает роль "Зарегистрированный пользователь"
- Создается профиль с базовыми данными
 
**Регистрация автора**
- Пользователь подает заявку на роль автора
- Заявка попадает в очередь модерации
- Модератор рассматривает заявку
- При одобрении пользователь получает роль "Автор"

**Регистрация модератора**
- Модератор является уже назначенной ролью для пользователей с определенными данными(логин, пароль)
