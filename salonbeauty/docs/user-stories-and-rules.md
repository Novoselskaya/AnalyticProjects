
## 1. Бизнес-требования

### 1.1. Исходные данные  и цель

Единая платформа для сети салонов красоты, которая предоставляет возможности организации рабочего расписания, запись для клиентов, ведение профилей мастеров и клиентов. 
#### Роли:

- **Администратор салона**: расписание, оплаты, акции.
- **Владелец**: отчет по продажам.
- **Мастер**: расписание, записи.
- **Клиент**: запись.


### 1.2 Возможности бизнеса

Для бизнеса мы хотим получить:
- большую клиентскую базу;
- единый учет записей по всем филиалам;
- единая политика для каждого филиала;
- рост повторных визитов.

**Привлекательность и отличия предлагаемого решения:** 
- единая предметная модель (салон, мастер, услуга, запись, акция, оплата);
- идемпотентность финансовых операций;
- возможность масштабирования сети и быстрого подключения нового филиала в работу;
- система акций и скидок;
- система отзывов клиента на услугу мастера;
- общий анализ всех доходов для администратора;
- техническая гарантия отсутствия пересечений в записи клиентов.



### 1.3 Бизнес-цели  

1. Увеличить выручку и загрузку: +X% выручки / +Y% заполненности за N месяцев.
2. Снизить операционные затраты и ошибки: −X% времени админов на ведение записей и предварительной переписки с клиентом.
3. Подготовить масштабирование сети: подключение нового салона в сеть в приложение за X дней.
4. Время записи клиента на сайте в среднем будет составлять X минут без взаимодействия сотрудника.
5. Повысить вероятность повторного визита клиента с помощью системы лояльности, доступной каждому.
6. Достигнуть показателя удовлетворения покупателей, равного по крайней мере X положительных отзывов на каждого мастера, в течение Y месяцев со времени выпуска продукта.
7. Заполнить расписание мастеров на X%. 
8. Перевести минимум X% клиентов в формат онлайн-записи.
  
### 1.4. Положение о концепции проекта

 - Для клиентов сети салонов красоты, которые нуждаются в удобной записи и управлением своими записями, в предоставлении полной информации о программе лояльности, ценах, мастерах, филиалах 
 - Для мастеров, которые нуждаются в сформированном заполненном расписании
 - Для администраторов, которые нуждаются в управлениями программой лояльности и отчете об оплатах
 - Для владельцев салона, которые нуждаются в своде отчетов
  приложение SalonNetwork является единой системой для организации возможностей клиента, мастеров, администратора и управляющего. Данная платформа обеспечивает онлайн-запись и управление ими, управлением расписанием мастеров, применение программы лояльности, отчетность по выручке, по посещениям салона, прием оплаты, а также отзывы клиентов. Преимуществом является единая сводка всех данных в одном месте, отсутствие конфликтов расписания, а также возможность сетевого роста на одной базе.
 

### 1.5. Бизнес-риски

- первое время клиенты не смогут сразу узнать о новой платформе и будут пользоваться устаревшими формами записи. Система не будет являться главным каналом для записи;
- плохая работа мастеров влечет за собой негативные отзывы клиентов, что подрывает репутацию салона;
- организация оплаты для поддержания идемпотентности финансовых операций;
- юридические риски по обработке персональных данных;

### 1.6. Предположения и зависимости

**Предположения :**
1. Система является основным источником информации о записях. Доля записи в системе будет значительно превышать долю записей во внешних каналах по началу. Далее для повышения и перевода еще большего количества людей в систему внедрить регламент записи только через систему. 
2. Услуги и их длительность можно расположить в расписании таким образом, чтобы расписание было максимально оптимизированным.
3. Должны существовать правила записи на визит, отмен и переноса записей(подробнее в документе Use case и бизнес-правила)

**Зависимости:**

1. Платёжный провайдер для поддержания правильности финансовых операций.
2. Правила работы с транзакциями для поддержания финансовой идемпотентности.
3. Канал уведомлений SMS и его доставка.
4. Поддержка механизма событий, их обработки и публикаций.
5. Требования по персональным данным.
